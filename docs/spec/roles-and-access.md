member
	•	現場ユーザー。
	•	appドメインで自分の業務データを登録・閲覧できる。
	•	他ユーザー管理や組織設定には触れられない。そういうUIは表示しない。

admin
	•	memberの権限をすべて持つ。
	•	組織の運用管理者。
	•	adminドメインで同一組織内ユーザーを管理できる（CRUD）。
	•	ユーザーの作成・更新・無効化。
	•	member / admin ロールの付与・剥奪。
	•	組織設定のうち、日常運用に関わる範囲は扱える（例：誰が使えるか）。
	•	ただし以下はできない：
	•	支払い情報の変更
	•	組織の凍結/廃止
	•	ownerロール関連（付け替え・譲渡など）

owner
	•	adminの権限をすべて持つ。
	•	組織の代表。1組織につき必ず1人。削除不可。譲渡のみ可。
	•	追加でできること：
	•	組織の支払い情報を変更できる。
	•	組織の廃止 / 凍結を指示できる（状態遷移させる）。
	•	admin権限の付け替えができる（誰をadminにするか決められる）。
	•	owner権限を他ユーザーに譲渡できる（自分は降格する）。
	•	上記の操作はすべて activity_logs に記録する。

ops
	•	SaaS提供側の運用担当。
	•	今回の雛形では未実装（UIもAPIもなし）。
	•	将来的に複数組織を横断してサポート・監査する想定。
